## ChassisFactory {#ChassisFactory}

`ChassisFactory` 类用于创建底盘实例。

- 方法原型：

  ```python
  create_chassis_from_config(config_file: str) -> ChassisInterface
  ```

- 参数说明：

  <table>
     <thead>
        <tr>
          <th><strong>参数名称</strong></th>
          <th><strong>说明</strong></th>
        </tr>
     </thead>  
     <tbody>
        <tr>
           <td>`config_path`</td>
           <td>
                配置文件路径。配置文件为 JSON 文件，包含以下键值对。

                 ```json
                 {
                    "chassis_type":"woosh",   # 底盘厂商
                    "chassis_name":"chassis", # 底盘名称
                    "ip":"127.0.0.1",         # IP 地址
                    "port":51051              # 端口号
                 }
                 ``` 
           </td>
        </tr>
     </tbody>
  </table>  

- 返回值：

  若创建成功，则返回 [ChassisInterface](#ChassisInterface) 实例；否则返回 `None`。

## ChassisFactoryParam {#ChassisFactoryParam}

`ChassisFactoryParam` 类用于描述创建底盘时所需的参数。

| 属性 | 数据类型 | 说明 |
|:---|:---|:---|
| `chassis_type` | str | 底盘厂商，例如 `"woosh"`。 |
| `chassis_name` | str | 底盘名称，例如 `"chassis"`。 |
| `ip` | str | IP 地址，例如 `"127.0.0.1"`。 |
| `port` | int | 端口号，例如 `54051`。 |

## ChassisInterface {#ChassisInterface}

`ChassisInterface` 类提供了一系列用于控制底盘的公共方法，例如移动底盘、获取轮廓和查询状态等。在使用前，请确保已创建底盘实例。

### 连接底盘

在创建底盘实例后，用户可通过如下方法与底盘建立连接。

- 方法原型：

  ```python
  connect(self) -> alphabot.core.AbcErrorCode
  ```

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。

### 断开底盘

用户可通过如下方法断开与底盘的连接。

## ChargeState {#ChargeState}

`ChargeState` 类是一个表示底盘电池充电状态的枚举器。

<Aside>
此类为枚举类，用户应直接使用类中定义的常量成员，无需自行实例化。
</Aside>

| 常量成员 | 值 | 说明 |
|:---|:---|:---|
| `UNDEFINED` | 0 | 未定义 |
| `NOT` | 1 | 未充电 |
| `MANUAL` | 2 | 手动充电中 |
| `AUTO` | 3 | 自动充电中 |
