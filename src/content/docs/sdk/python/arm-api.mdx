## ArmFactory {#ArmFactory}

`ArmFactory` 类提供了如下方法用于创建机械臂实例。

1. 根据机械臂的厂商和名称，创建机械臂实例

   - 方法原型：
   
     ```python
     create_arm(vendor_type: str, name: str, params: dict[str, str]) -> ArmInterface
     ```

   - 参数说明：

     <table>
     <thead>
     <tr>
     <th><strong>参数名称</strong></th>
     <th><strong>说明</strong></th>
     </tr>
     </thead>
     <tbody>
     <tr>
     <td>
     `vendor_type`
     </td>
     <td>     
     机械臂厂商，如 `"realman"`。     
     </td>
     </tr>
     <tr>
     <td>
     `name`
     </td>
     <td>
     机械臂名称，如 `"left_arm"` 或 `"right_arm"`。
     </td>
     </tr>
     <tr>
     <td>
     
     `params`
     
     </td>
     <td>
     
     机械臂其他参数，如 IP 地址、端口等。
     
     ```json
     {
        'ip':'192.168.1.10',      # TCP IP 地址
        'port':'5000',            # TCP 端口号
        'udp_ip':'192.168.1.100', # UDP IP 地址
        'udp_port':'6000',        # UDP端口号
        'udp_cycle':'20', # UDP 周期，以 udp_cycle*5 毫秒为周期
     }
     ```
     
     </td>
     </tr>
     </tbody>
     </table>


    - 返回值：
    
      若创建成功，则返回 [ArmInterface](#ArmInterface) 实例；否则返回 `None`。

2. 根据 ArmFactoryParam 对象，创建机械臂实例

   - 方法原型：
   
     ```python
     create_arm(param: ArmFactoryParam) -> ArmInterface
     ```

   - 参数说明：
   
     | 参数名称 | 说明 |
     |:---|:---|
     | `param` | 机械臂相关参数，具体参见 [ArmFactoryParam](#ArmFactoryParam)。 |

   - 返回值：
   
     若创建成功，则返回 [ArmInterface](#ArmInterface) 实例；否则返回 `None`。

3. 根据配置文件，创建机械臂实例

   - 方法原型：
   
     ```python
     create_arm_from_config(config_path: str) -> ArmInterface
     ```

    - 参数说明：

      <thead>
      <tr>

      <th><strong>说明</strong></th>
      </tr>
      </thead>
      <tbody>
      <tr>
      <td>
      
      `config_path`
      
      </td>
      <td>
      
      配置文件路径。配置文件为 JSON 文件，包含以下键值对。
      
      ```json
      {
          "arm_type":"realman", # 机械臂厂商
          "arm_name":"left_arm", # 机械臂名称
          "ip": "192.168.1.18", # TCP IP 地址
          "port": "8080", # TCP 端口号
          "udp_ip": "192.168.1.102", # UDP IP 地址
          "udp_port": "8089", # UDP 端口号
      ```     

## ArmInterface {#ArmInterface}

`ArmInterface` 类提供了一系列用于控制机械臂的公共方法，例如移动、抓取和状态查询。在使用前，请确保已创建机械臂实例。

### 属性

| 属性（只读） | 数据类型 | 说明 |
|:---|:---|:---|
| `name` | str | 机械臂名称 |

- 方法原型：

  ```python
  connect(self) -> alphabot.core.AbcErrorCode
  ```

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。

### 断开机械臂

- 方法原型：


- 返回值：

  `None`。

### 关节空间运动

- 方法原型：

  ```python
  move_joint_position(
      self,
      joints: list[float],
      v: int,
      r: int,
      trajectory_connect: int,
      block: int
  ) -> alphabot.core.AbcErrorCode
  ```

- 参数说明：

  | 参数名称 | 说明 |
  |:---|:---|
  | `joints` | 目标关节位置，单位为角度。 |
  | `v` | 规划速度和加速度占关节最大线转速和加速度的百分比，取值范围为 0~100。 |
  | `r` | 轨迹交融半径，默认值为 0。 |
  | `trajectory_connect` | 轨迹连接标志，取值为 0 或 1。<br/>- `0`：立即规划并执行轨迹，不与后续轨迹连接。<br/>- `1`：将当前轨迹与下一条轨迹一起规划，但不立即执行。注意：在阻塞模式下，即使运动轨迹发送成功，函数也会立即返回。 |
  | `block` | 设置阻塞模式，取值为 0 或 1。<br/>- `0`：非阻塞模式。函数在成功将运动轨迹发送至控制器后立即返回。<br/>- `1`：阻塞模式。函数将等待，直到机械臂完成运动或发生错误后才返回。 |

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。

### 角度透传

角度通过 CAN FD（Controller Area Network Flexible Data-rate）透传给机械臂，不需控制器规划。若调用正确，机械臂立即执行。

<Aside>
- 透传效果受通信周期和轨迹平滑度影响，因此要求通信周期稳定，避免大幅波动。
- 用户在使用此功能时，建议进行良好的轨迹规划，以确保机械臂的稳定运行。
- I 系列有线网口周期最快可达 2ms，提供了更高的实时性。
</Aside>

- 方法原型：

  ```python
  move_joint_position_canfd(
      self,
      joints: list[float],
      follow: bool,
      trajectory_mode: int = 0,
      radio: int = 0
  ) -> alphabot.core.AbcErrorCode
  ```

- 参数说明：

  | 参数名称 | 说明 |
  |:---|:---|
  | `joints` | 目标关节位置，单位为角度。 |
  | `follow` | 设置跟随模式，取值为 `True` 或 `False`。<br/>- `True`：高跟随。<br/>- `False`：低跟随。<br/>若使用高跟随，透传周期要求不超过 10ms。 |
  | `trajectory_mode` | 在**高跟随模式**下使用，取值为 0、1 或 2。<br/>- `0`：完全透传模式。<br/>- `1`：曲线拟合模式。<br/>- `2`：滤波模式。 |
  | `radio` | - 曲线拟合模式：radio 是平滑系数，取值范围为 0\~100。<br/>- 滤波模式：radio 是滤波参数，取值范围为 0\~1000。 |

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。

### 位姿透传

位姿通过 CAN FD（Controller Area Network Flexible Data-rate）透传给机械臂，不需控制器规划。

当目标位姿被透传到机械臂控制器时，控制器首先尝试进行逆解计算。若逆解成功且计算出的各关节角度与当前角度差异不大，则直接下发至关节执行，跳过额外的轨迹规划步骤。

这一特性适用于需要周期性调整位姿的场景，如视觉伺服等应用。

<Aside>
- 透传效果受通信周期和轨迹平滑度影响，因此要求通信周期稳定，避免大幅波动。
- 用户在使用此功能时，建议进行良好的轨迹规划，以确保机械臂的稳定运行。
- I 系列有线网口周期最快可达 2ms，提供了更高的实时性。
</Aside>

- 方法原型：

  ```python
  move_cartesian_pose_canfd(
      self,
      pose: alphabot.core.Pose,
      follow: bool,
      trajectory_mode: int = 0,
      radio: int = 0
  ) -> alphabot.core.AbcErrorCode
  ```

- 参数说明：

  | 参数名称 | 说明 |
  |:---|:---|
  | `pose` | 目标位姿。 |
  | `follow` | 设置跟随模式，取值为 `True` 或 `False`。<br/>- `True`：高跟随。<br/>- `False`：低跟随。<br/>若使用高跟随，透传周期要求不超过 10ms。 |
  | `trajectory_mode` | 在**高跟随模式**下使用，取值为 0、1 或 2。<br/>- `0`：完全透传模式。<br/>- `1`：曲线拟合模式。<br/>- `2`：滤波模式。 |
  | `radio` | - 曲线拟合模式：radio 是平滑系数，取值范围为 0\~100。<br/>- 滤波模式：radio 是滤波参数，取值范围为 0\~1000。 |

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。

### 松开夹爪

夹爪以指定的速度运动到开口最大处。


### 配置 Modbus RTU 模式

在机械臂启动后，用户必须先调用如下方法配置 Modbus RTU 模式，才可对通信端口进行操作，否则会返回错误信息。另外，机械臂将会对用户的配置方式进行保存，机械臂重启后会自动恢复到用户断电之前配置的模式。

- 方法原型：

  ```python
  set_modbus_mode(
      self,
      port: int,
      baudrate: int,
      timeout: int
  ) -> alphabot.core.AbcErrorCode
  ```

- 参数说明：

  | 参数名称 | 说明 |
  |:---|:---|
  | `port` | 指定通信端口，取值为 0、1 或 2。<br/>- `0`：控制器 RS485 端口为 RTU 主站。<br/>- `1`：末端接口板 RS485 接口为 RTU 主站。<br/>- `2`：控制器 RS485 端口为 RTU 从站。 |
  | `baudrate` | 波特率，支持 9600、115200 和 460800 三种常见波特率。 |
  | `timeout` | 超时时间，单位百毫秒。对 Modbus 设备所有的读写指令，在规定的超时时间内未返回响应数据，则返回超时错误提示信息。超时时间不能为 0，若设置为 0，则机械臂按通信端口 1 进行配置。 |

- 返回值：

  `AbcErrorCode.SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/python/data-structures#AbcErrorCode)。
