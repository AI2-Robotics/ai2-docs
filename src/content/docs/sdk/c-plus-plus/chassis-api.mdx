底盘相关的接口由如下两个类实现。您可以在 `c++\include\drivers\chassis` 目录下找到对应的头文件以获取完整 API。

| 头文件 | 类名 | 说明 |
|:------|:----|:-------|
| `chassis_factory.h` | `ChassisFactory` | 底盘工厂类，用于创建不同类型的底盘实例。 |
| `chassis_interface.h` | `ChassisInterface` | 底盘接口基类，定义了底盘控制的通用接口。所有特定厂商的底盘驱动都应该继承此类。 |

## 创建底盘

`ChassisFactory` 类提供了两种方式创建底盘实例，具体如下。

1. 根据底盘参数，创建底盘实例。

   - 方法原型：

      ```cpp
      static std::shared_ptr<ChassisInterface> create_chassis_from_config(
        const std::string& config_path)
       ```

   - 参数说明：

     | 参数名 | 类型 | 说明 |
     |:-------|:---|:-|
     | `config_path` | 输入参数 | 底盘相关参数，具体参见 [ChassisFactoryParam](/sdk/c-plus-plus/data-structures#ChassisFactoryParam)。 |

   - 返回值：

     若创建成功，则返回 `ChassisInterface` 类型的共享智能指针；否则返回 `nullptr`。

2. 根据配置文件，创建底盘实例。

   - 方法原型：

      ```cpp
      static std::shared_ptr<ChassisInterface> create_chassis_from_config(
        const std::string& config_path)
       ```

   - 参数说明：

     <table>
     <thead>
     <tr>
     <th><strong>参数名称</strong></th>
     <th><strong>类型</strong></th>
     <th><strong>说明</strong></th>
     </tr>
     </thead>
     <tbody>
     <tr>
     <td>
     
     `config_path`
     
     </td>
     <td>
     
     输入参数
     
     </td>
     <td>
     
     配置文件路径。配置文件为 JSON 文件，包含以下键值对。
     
     ```json
     {
        "chassis_type": "woosh", # 底盘厂商
        "chassis_name": "chassis", # 底盘名称
        "ip": "127.0.0.1", # IP 地址
        "port": 51051 # 端口号
     }
     ```
     
     </td>
     </tr>
     </tbody>
     </table>

   - 返回值：

     若创建成功，则返回 `ChassisInterface` 类型的共享智能指针；否则返回 `nullptr`。 

## 连接底盘

在创建底盘实例后，您可通过如下方法与底盘建立连接。

- 方法原型：

  ```cpp
  AbcErrorCode connect()
  ```
- 返回值：

  `AbcErrorCode::SUCCESS` 表示成功；其他错误码，请参见 [AbcErrorCode](/sdk/c-plus-plus/data-structures#AbcErrorCode)。
