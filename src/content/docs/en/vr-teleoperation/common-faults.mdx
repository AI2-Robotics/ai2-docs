## VR Controllers Cannot Control the Robotic Arm

* Cause:

    1. VR USB cable is not connected.
    2. VR USB permission is not granted.
    3. VR USB debugging permission is not granted.

* Solution:
    
    1. Check whether the VR USB cable is properly connected.
    2. Reconnect the USB cable, and reconfigure VR USB permission according to the [Prerequisites](../operation-guide/#preconfiguration).
    3. On the robot terminal, run `rm -rf .android` command, then reconnect the USB cable.


## Other Problems

If you encounter other problems, please first try restarting the device. If the problem still exists, please contact our support team.
