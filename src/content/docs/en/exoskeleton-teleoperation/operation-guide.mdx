Mastering the following operations will help you use the Exoskeleton Teleoperation Device more efficiently.

## Starting the HMI Interface{#start-hmi-interface}

ROBOT-CONTROL-HMI (hereinafter referred to as HMI) is the graphical user interface for controlling and monitoring the robot system. It serves as the core tool for interacting with the robot when using the Exoskeleton Teleoperation Device. The HMI consists of two separate programs: the user-side program and the robot-side program. Both must be started before use, following the procedures below.

### Starting the User-Side Program

import operation_guide_1 from 'img/operation_guide_1.png';

<Steps>

1. On the industrial PC, download the [Microsoft Edge browser installation package](https://www.microsoft.com/en-us/edge/download).

2. In the folder containing the installer package, open a terminal and run the following command to install the browser.

   ```bash
   sudo dpkg -i microsoft-edge-stable_xxx_arm64.deb
   ```
   <Aside>
   Replace `microsoft-edge-stable_xxx_arm64.deb` in the command with the actual filename of the installation package.
   </Aside>


3. In the Ubuntu operating system, navigate to `Settings -> Default Applications -> Web (W)` , select Microsoft Edge, and set Microsoft Edge as the default browser.

4. On the industrial PC, open a terminal and enter the following commands.

   ```bash
   cd ~
   bash ~/user_docker_start_xxx.sh
   ```
   <Aside>
   `xxx` indicates the specific version number.
   </Aside>

5. After the script executes, the Microsoft Edge browser will launch the HMI interface automatically. If it does not open automatically, manually enter `http://127.0.0.1:4090` in the browser's address bar.

   <Image src={operation_guide_1} alt="" />

   <Aside>
   - The Exoskeleton Device, VR Control Device, and Active Exoskeleton Device are three independent systems. Therefore, when switching between starting the Exoskeleton Device, VR Control Device, and Active Exoskeleton Device, you must close all controls running under the previous device first. Ensure all buttons in the Robot Control panel are blue before switching devices.
   - To close the HMI, first press `Ctrl+C` to terminate both the user-side and robot-side programs, then close the corresponding terminals, and finally close the Microsoft Edge browser.
   </Aside>
   <Aside type="tip">
      To maintain communication, the user-side program requests and updates robot status data approximately every 10 seconds. With both programs running, periodic robot status retrieval may fail only in the following two scenarios:
      - HMI interface closed: The user-side program cannot retrieve robot status via the browser if the interface is manually closed with no other interface active.
      - HMI interface suspended by browser background: In multi-tab browsing, if the HMI interface loses focus, the browser may automatically suspend it, preventing the user-side program from sending status requests.
   </Aside>
       

</Steps>

### Starting the Robot-Side Program

<Steps>

1. On the industrial PC, open a new terminal window and remotely connect to the robot terminal using the `ssh` command.

   ```bash
   zpf@zpf-MIC-770V3H:~$ ssh robot@192.168.1.100
   # Password: robot
   robot@192.168.1.100's password: 
   # Connection is successful if the terminal username starts with "robot".
   robot@ter30jb3-ubuntu:~$ 
   ```

2. In the successfully connected terminal, enter the following commands.

   ```bash
   cd ~
   bash ~/robot_docker_start.sh
   ```
   <Aside>
   To close the HMI, first press `Ctrl+C` to terminate both the user-side and robot-side programs, then close the corresponding terminals, and finally close the Microsoft Edge browser.
   </Aside>
</Steps>

After Starting the HMI interface, you can [use the exoskeleton device](#using-exoskeleton-device) to control the robot.

## Using the Exoskeleton Device{#using-exoskeleton-device}

### Starting the Exoskeleton Control Device{#start-exoskeleton-device}

import operation_guide_2 from 'img/operation_guide_2.png';
import operation_guide_3 from 'img/operation_guide_3.png';

<Steps>
1. After [starting the HMI interface](#start-hmi-interface), click the `Start Exoskeleton Device` button. Once started successfully, the button will turn red and the text will change to `Stop Exoskeleton Device`.
   <Image src={operation_guide_2} alt="" />
   <Aside>
   - The Exoskeleton Device, VR Control Device, and Active Exoskeleton Device are three independent systems. Therefore, when switching between starting the Exoskeleton Device, VR Control Device, and Active Exoskeleton Device, you must close all controls running under the previous device first. Ensure all buttons in the Robot Control panel are blue before switching devices.
   - If the robot malfunctions or the robotic arm moves abnormally or becomes uncontrollable, click the `Stop Exoskeleton Device` button immediately. The robot will emergency stop and cease response.
   </Aside>

2. The AI2-HMI window will pop up. This window displays real-time teleoperation data. Do not close this window during operation.

   <Image src={operation_guide_3} alt="" />
   
   <Aside>
   - If there is only one monitor, minimize this window to ensure it does not affect the operation of the main HMI interface. If there are multiple monitors, drag this window to a non-main monitor for easy observation of data changes.
   - This window does not automatically exit when the HMI interface is closed. It can only be closed by clicking its own close button or the `Stop Exoskeleton Device` button on the HMI interface.
   - To prevent display abnormalities or data conflicts caused by multiple windows running simultaneously, ensure only one HMI interface and one corresponding AI2-HMI window are open before each startup.
   </Aside>

</Steps>

### Starting Follow-Up{#start-follow}

import operation_guide_7 from 'img/operation_guide_7.png';
import operation_guide_8 from 'img/operation_guide_8.png';
import operation_guide_9 from 'img/operation_guide_9.png';

<Steps>

   <Image src={operation_guide_7} alt="" />

2. A prompt message as shown below will appear. The robotic arm will slowly move to the pose of the teleoperation arm. After confirming that the pose is normal, click the `OK` button to switch to fast follow-up.
   <Aside type="tip">
   In slow follow-up mode, the robotic arm follows the movement of the teleoperation arm at a slow speed, allowing the operator to observe the motion status of the robotic arm. After confirming that the motion pose is consistent with the teleoperation arm and there are no abnormalities, switch to fast follow-up for a smoother operation experience.
   </Aside>

   <Image src={operation_guide_8} alt="" width="450" />

3. Click the `OK` button in the pop-up window below to switch to Fast Follow-Up.
   <Image src={operation_guide_9} alt="" width="450"/>

4. Once started successfully, the blue Start `Slow Follow-Up` button will change to the red `Stop Fast Follow-Up` button. Also confirm that the follow-up status in the `System Status` panel is `Fast Follow-Up`.

</Steps>

## Using the Controller

import operation_guide_22 from 'img/operation_guide_22.png';
import operation_guide_23 from 'img/operation_guide_23.png';

After [starting the exoskeleton device](#start-exoskeleton-device), you can use the controller to control the robot.

<Steps>

1. Ensure that the controller receiver is connected to the USB port of the industrial PC and that the controller has sufficient battery power.

   <Aside>
   - Do not connect the controller to the industrial PC using a wired connection.
   - The controller charging specification is 5V/1-2A. Fast charging is not supported. Do not use a fast charger with a power output exceeding 66W. The charging status indicators are described below.

      <Image src={operation_guide_22} alt="" width="450"/>
   </Aside>

2. When using the controller for the first time, the controller mode must be configured to `Android Mode -> Receiver` Mode.

   Configuration method: Press and hold the HOME button for 3 seconds to enter the mode selection interface, then select `Android Mode -> Receiver`.

3. On the industrial PC, open a terminal and enter the following command:

   ```bash
   ros2 run joy joy_node
   ```

   <Aside>
   Do not close this terminal window while using the controller, otherwise the controller will become unusable.
   </Aside>
   
4. You can control the neck, waist-legs, and chassis respectively through the controller buttons.

   <Image src={operation_guide_23} alt="" width="500"/>

         <tr>

         </tr>
      </thead>

            </td>
            <td>      
               - Move forward/backward: LB + Left Stick Y
               - Turn left/right: LB + Left Stick X
            </td>
         </tr>
         <tr>
            <td>
            Control Neck
            </td>
            <td>      
               - Turn head left/right: Right Stick X
               - Tilt head up/down: Right Stick Y
            </td>
         </tr>

            <td>

               - Raise waist/legs: RT + A

               - Rotate waist left: LT + X
               - Move waist forward/backward: Left Stick Y
            </td>
         </tr>       
      </tbody>
   </table>   

</Steps>



## Data Acquisition

After [starting the exoskeleton device](#start-exoskeleton-device), you can acquire data from the Exoskeleton Teleoperation Device using the `Robot Data Acquisition` function in the HMI interface.


<Aside>
The Exoskeleton Device, VR Control Device, and Active Exoskeleton Device are three independent systems; do not mix them.
</Aside>

<Steps>

   <Image src={operation_guide_47} alt="" />

## Zero Calibration

The zero calibration operation is only required after disassembling or assembling the structure. When the pose of the robotic arm is inconsistent with that of the teleoperation arm, and no over-limit is displayed on the AI2-HMI interface, it may be a zero position abnormality. Please contact our after-sales engineers for assistance.
