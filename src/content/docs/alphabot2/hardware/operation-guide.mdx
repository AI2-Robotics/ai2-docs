## 急停

import architecture_12 from 'img/architecture-12.png';

<Aside type="caution">
- 严禁遮挡、损坏或拆除急停装置。
- 每日开机前须检查急停装置：外观无异常，按钮按下及复位功能正常。
</Aside>

若机器人发生或可能发生危险或异常时，请立即按下后背急停按钮，机器人将立即停止并进入安全停机状态，具体操作如下。

<Steps>
1. 用力按下后背红色按钮，听到“咔哒”声，表示触发急停。

   <Image src={architecture_12} alt="" width="400"/>

2. 确认机器人是否完全静止，本体及末端执行器无任何运动。
3. 检查底盘指示灯。若红色灯闪烁，表示急停生效。
   <Aside>
    - 急停生效后，机器人将切断电机驱动信号，进入安全停机状态。此时机器人无法执行任何运动指令，但仍然供电以保持通信和状态监测。
    - 急停生效后，请排查异常或危险源（包括清空运动区域内的人员或异物）。如有需要，请切断相关故障设备的电源。
   </Aside>
4. 若需要恢复机器人使用，请顺时针旋转红色按钮，直至其弹出（听到“弹起”声）。

</Steps>

## 充电

AlphaBot 2 内置电池电量不足时，会语音提示需要充电。AlphaBot 2 提供了如下两种充电方式，请根据具体场景选择合适的充电方式。

1. 使用电源适配器充电：机器人关机或切换手动模式后，使用电源适配器直接对机器人进行手动充电。
   <Image src={architecture_9} alt="" width="300"/>
   
2. 使用充电箱充电：机器人自行移动到充电箱进行自充电。
   <Image src={architecture_13} alt="" width="400"/>

在开始以下操作前，请阅读以下安全注意事项。

<Aside type="caution">
- 为获得最佳效能并避免兼容性问题，请使用产品随附的电源适配器、充电箱及电源线。
- 严禁混用不同产品的电源适配器、充电箱，以免造成充电异常或损坏设备。
- 请务必使用本司配置的电源适配器、充电箱，如使用其他劣质充电器造成任何损失，我司概不负责。
- 连接电源适配器或充电箱时，请确保所有端口（包括电源线、变压器和插座）均已完全推入并牢固连接，以避免因松脱而导致充电不良等问题的发生。
- 尽量勿将电源适配器及电源线置于地面，以防办公椅轮组压损线缆导致接触不良或漏电。
- 电源线应尽量展开，切勿缠绕于电源适配器或其他物品上。
- 充电过程中请勿移动机器人，以免造成充电接口损坏或安全事故。
- 充电完成后请及时断开电源适配器或充电箱，避免过度充电对电池寿命造成影响。
- 若电源适配器、充电箱或机器人出现异常，请立即停止使用并切断电源。
</Aside>

## 远程连接主控板{#remote_connect}

在某些情况下（如调试），电脑需要通过 ssh 命令远程连接至主控板。主控板的信息如下表所示。

|信息项|说明|
|---|---|
|主控模块|NVIDIA AGX Orin 64G|
|用户名|robot|
|IP 地址|192.168.90.1|
|Wi-Fi 名称|形如 Alphabot2-xxx-xxxx，具体见交付清单|
|Wi-Fi 初始密码|88888888|

接下来以 Windows 操作系统为例，详细讲解电脑如何连接主控板。

<Steps>

1. 点击无线网，查找形如 `Alphabot2-xxx-xxxx` 的 Wi-Fi 网络名称，然后点击 `连接`。此处以 Alphabot2-088-5d50 为例。
   <Image src={operation_8} alt=""/>

2. 在搜索框中搜索 `网络连接`，点击 `查看网络连接`。
   <Image src={operation_9} alt="" width="500"/>

3. 双击 `Alphabot2-xxx-xxxx`，点击 `详细信息`，查看其 IPv4 默认网关地址。该地址即为远程登陆主控板的 IP 地址。
   <Image src={operation_10} alt="" />

4. 使用快捷键 `Win+X`，弹出菜单，点击 `终端`。
   <Image src={operation_11} alt=""/>

5. 在终端输入 `ping 192.168.90.1`，显示下图则表示，可进行登录操作。
   <Image src={operation_12} alt=""/>

6. 在终端输入 `ssh robot@192.168.90.1`，随后输入 `yes`，以及登录密码（默认初始密码为 `88888888`），显示下图则表示连接主控板成功。
   <Image src={operation_13} alt=""/>
   
</Steps>

若需配置 Wi-Fi 连接，请继续以下操作。

<Steps>

1. 在上一步的终端中继续输入 `sudo nmcli device wifi list` 命令，以查看当前可用的 Wi-Fi，并记录要连接的 Wi-Fi 的名称（SSID）。
   <Image src={operation_14} alt=""/>

2. 在上一步的终端中继续输入 `sudo nmcli device wifi connect <SSID> password <password>` 命令，以连接新的 Wi-Fi，其中 `<SSID>` 和 `<password>` 分别替换为要连接的 Wi-Fi 的名称和密码。
   <Image src={operation_15} alt=""/>

3. 输入 `ifconfig` 命令以查看无线网络分配给主控版的 IP 地址（即 `wlan0/wlp2s0` IP 地址），以检查并确认是否已经成功连接到指定的 Wi-Fi 网络，自此 Wi-Fi 配置完成。
   <Image src={operation_16} alt=""/>

4. 完成 Wi-Fi 配置后，电脑连接上述 Wi-Fi，然后在终端输入命令 `ssh robot@x.x.x.x`，其中 `x.x.x.x` 为上一步获取的 IP 地址，随后输入 `yes`，以及登录密码，登录成功。每次连接主控板时，无需再配置 Wi-Fi。
   <Image src={operation_17} alt=""/>

</Steps>
